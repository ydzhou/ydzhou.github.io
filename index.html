Parallel Computing on a Jacobian Solver

Paralle computing is a quit interesting way to distribute the workload among all the available workers, then significantly reduce the run-time relying on their teamwork. It is a smart way to tackle performance issue when we do not have high efficiency on a single processing unit. The Jacobian solver apply a update equation in each iteration, and does not require data from previous iterations. Therefore, we can get the value of the next iteration, based on that of current iteration. This property allows us to do the parallel computing within each iteration.

I have used MPI technique to implement a parallelized Jacobian Solver. Implemented Jacobian Solver is used to conduct simulation to estimate properties of a large-scale reservoir. The performance gain is significant, thanks to the power of parallel computing. With the size of matrix increasing, the overhead of communications between each processor has less impact on performance. It is very practical to apply in the real-world problem.

A x86 Operating System Implementation

It is built from scratch. It contains basic memory management, including virtual memory, paging and frame allocation. It has a schedular to handle multiple threads running in parallel. It also supports a file system along with a blocking disk driver.

The implementation fully functions in Bochs and demonstrates the basic concepts referred to operating system.
