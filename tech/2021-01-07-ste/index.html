A simple terminal text editor written in Go

I wrote a simple text editor on the terminal.

Overview

A text editor has three modules:

  1. controller: to handle customers’ keyboard inputs and to dispatch for further processing.
  2. model: to store what customers are working on and to process necessary business logic.
  3. view: to render the user interface.

The text editor runs in an infinite loop. First we clear the screen and render the document content to stdout. Which part of content to display is decided by current position of the cursor and the size of the terminal screen. We then listen on the key input at stdin so that we can process Users’ commands.

for {
    editor.display.DrawScreen(e.buf, e.cursorX, e.cursorY)
    if editor.process() {
        break
    }
}

How to Handle Key Input

We listen to Stdin per rune. A regular key takes one rune, thus a single input will be sent to processing once captured. The special key, also called the escape key, takes multiple runes. The escape key always starts with rune of value 27. Therefore if we listen to a rune of value 27, we will continue reading the following runes in Stdin. For example, if an arrow up key is pressed, it will send three runes altogether. We need to read them all. The third rune of value 65 will give its meaning. Once we figure out which key a customer has pressed, we can continue either putting regular keys into the document or executing special actions such as move cursors around or quitting the editor.

if r == 27 {
    if e.reader.Buffered() == 0 {
		return ESCAPE
    }
    for i := 0; i < 3; i++ {
        r, _, err = e.reader.ReadRune()
        if err != nil {
            return UNKNOWN
        }
        if i == 1 {
            switch r {
            case 65: return ARROW_UP
            case 66: return ARROW_DOWN
            case 67: return ARROW_case
            RIGHT 68: return ARROW_LEFT
            }
        }
    }
}

How to Store and Modify Document

We need some ways to store the text that is being processed. For the simiplicity, I use a single “big” string to store the text. I am going through how this string changes when users edit the document..

Insert a word

If a user types a word at the current cursor, this word is inserted in the cursor position. This can be done by adding the word to the string. Since cursor position is a 2D coordinate, we need to convert it into the corresponding index of our string as below:

func (b *Buffer) getIdx(x, y int) int {
    idx := 0
    for idx, _ = range b.txt {
        if x == 0 && y == 0 {
            break
        }
        if x > 0 && b.txt[idx] == rune('\n') {
            x--
            continue
        }
        if x == 0 {
            y--
        }
        if b.txt[idx] == rune('\n') {
            break
        }
    }
    return idx
}

Insert a new line

Inserting a new line is similar to insert a word by adding \n to the corresponding index.

Insert a tab

Tab is quite tricky because it is hard to determine its index just by the cursor position. One way is to convert the tab to spaces and store them in the buffer. This can break some file if their compiler rely on tab to process the code, such as Fortran or Assemble. Another way is to store tab as \t and then forbid cursor from moving in-between those spaces.

Delete a word

Similar to insertion, we can simply shrink the string to drop word at corresponding position. We do not need to worry about deleting the newline since we consider it as a regular word.

Move cursor around

We track cursor position in every loop so that we can do insertion or deletion at the correct index. One thing to notice is that, we need to fix the position if the user moves the cursor out of bound. For example, if the user tries to move the cursor to the end of line, the cursor need to be re-positioned to the beginning of next line. Besides that, when deleting a line breaker, we need to move the cusor to the end of previous line, and inserting a new line, the cursor need to be at the beginning of the next line.

Next we discuss how to display documents and handle scrolling.

How to Display Document

The editor view is rendered by converting our text object, a list of runes into a document under the 2D view. We render special character such as tab and line breaker in a human-readable way. If the current terminal window cannot fit the whole document, we also need to determine which part of the document need to be rendered, and we allow users to scroll around the document to read the rest. We do not support line wrapping.

Scrolling

We keep tracking two variables offsetX and offsetY. So the current viewable area is [offsetX, offsetX + viewX, offsetY, offsetY + viewY]. viewX and viewY are the length and width of current terminal. If the cursor is moving outside of this area, we need to update offsetX and offsetY.

func (d *Display) scroll(cursorX int, cursorY int) {
    if cursorX >= d.offsetX+d.viewX {
        d.offsetX = cursorX - d.viewX + 1
    } else if cursorX < d.offsetX {
        d.offsetX = cursorX
    }
    if cursorY >= d.offsetY+d.viewY {
        d.offsetY = cursorY - d.viewY + 1
    } else if cursorY < d.offsetY {
        d.offsetY = cursorY
    }
}

Then we can use offsetX to get viewX number of lines to display and for each line, we only display character after index offsetY.

Learning and Findings

  1. Go has native support to process UTF8 by using runes.
  2. Terminal control in Go need to be tailored for each operating system. For example, control flags to access term of OSX (freebsd) and Linux are different.
  3. Mapping 2D position in view and 1D index in text buffer (string) can be quite tedious.
